## EC2 とは

EC2 とは Amazon Elastic Compute Cloud の略で、AWS が提供する仮想マシンのサービスである。

仮想マシンとは、仮想化の技術を用いて、物理的なサーバー機器上で仮想的に動かす別のマシンのことをいう。

### 仮想マシン

仮想マシンを作成する方法は大きく分けて 2 種類ある。

- ホスト OS 型
- ハイパーバイザー型

#### ホスト OS 型

仮想化ソフトを導入し、ホスト OS の上にゲスト OS を導入していく方法を`ホストOS型`という。

![イメージ図](/img/svg/AWS/EC2/EC2-1.drawio.svg "ホストOS型")

##### メリット

- 既存の PC をそのまま用いることができる

##### デメリット

- ホスト OS とゲスト OS で処理を行う際に処理が遅くなる
- ハードウェアにアクセスする際にホスト OS を介するため、動作が遅い

#### ハイパーバイザー型

ホスト OS をインストールするのではなく仮想化ソフトを直接インストールし、その上にゲスト OS を導入していく方法を`ハイパーバイザー型`という。

![イメージ図](/img/svg/AWS/EC2/EC2-2.drawio.svg "ハイパーバイザー型")

##### メリット

- ハードウェアに直接アクセスできる

##### デメリット

- ハードウェアのスペックが低いと性能が期待できない

### コンテナ

仮想マシンに似たものとしてコンテナ技術がある。
コンテナ技術は仮想化マシンのように OS で分離するのではなく、アプリケーション単位で分離する技術である。
コンテナ技術を行うために使用するアプリケーションとして、Docker が有名である。

![イメージ図](/img/svg/AWS/EC2/EC2-3.drawio.svg "コンテナ")

##### メリット

- 起動・停止を高速で行うことができる
- コピーを作成するのが容易

##### デメリット

- コンテナを複数個運用した際に管理が大変
- 学習コストが高い

## EC2 が提供しているもの

EC2 はハイパーバイザー型のゲスト OS 以上の層を提供しているサービスで、1 つのゲスト OS を`EC2インスタンス`と呼ぶ。

ハイパーバイザー層以下は AWS が管理しているので、使用者側は管理する必要がないのも特徴の一つ。
![イメージ図](/img/svg/AWS/EC2/EC2-4.drawio.svg "EC2インスタンス")

### EC2 の特徴

様々な仮想環境を作成することができる。

- 汎用
- コンピューティング最適化
- ストレージ最適化
- メモリ最適化
- 高速コンピューティング(GPU)
- CPU の種類
- マシンの世代
- OS の種類

など用途に合わせて様々な環境を構築することができる。

EC2 は AWS が提供する幅広い種類の中からマシンイメージ(AMI : Amazon Machine Image)を選択して構築することが可能であり、普段使用している環境をインストールなし利用ができる。
さらに、AWS Marketplace を用いることで、ソフトウェアを含んだ環境を AMI から構築することも可能である。
